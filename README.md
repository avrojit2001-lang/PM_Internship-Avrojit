# PM_Internship-Avrojit

This repository stores the internship project deliverables and supporting PM documents organized by purpose.

## Folder structure
- documents/BRD — business requirements and baseline scope context
- documents/PRD — product requirements and solution intent
- documents/Plans — planning and scheduling artifacts
- documents/Reports — status and progress reporting
- documents/Registers — risk, RAID, and document register records
- documents/Notes — short notes and working observations
- documents/Logs — daily logs and activity records

## Versioning rules
- Use v1.0 for the first baseline release of a document.
- Use v1.1 for minor updates, clarifications, or corrections.
- Use v2.0 for major revisions that change scope, assumptions, or governance approach.

## Baseline vs. living documents
- Baseline documents capture approved scope, objectives, and requirements at a point in time.
- Living documents are updated as work evolves, such as plans, reports, risks, and logs.
